# Contributing to DB2 Genius Hub Lab

Thank you for your interest in improving this lab!

## How to Contribute

### Reporting Issues

If you find errors or have suggestions:
1. Check if the issue already exists
2. Create a new issue with:
   - Clear description of the problem
   - Steps to reproduce (if applicable)
   - Suggested fix (if you have one)

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/improvement`)
3. Make your changes
4. Test your changes thoroughly
5. Commit with clear messages (`git commit -m 'Add: new feature'`)
6. Push to your fork (`git push origin feature/improvement`)
7. Create a Pull Request

### Style Guidelines

- Use clear, concise language
- Include screenshots for UI changes
- Test all commands and code examples
- Follow existing markdown formatting
- Keep images in the `images/` directory

### Image Guidelines

- Use descriptive filenames (e.g., `GH-homepage.png`)
- Optimize images for web (reasonable file size)
- Use relative paths: `images/filename.png`
- Include alt text for accessibility

## Questions?

Contact the DB2 Genius Hub team for questions about contributing.
